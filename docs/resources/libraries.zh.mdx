# 库

Valtio 为代理状态管理提供了基本必需品，这对大多数项目来说都很棒；但是，一些用户希望扩展库的功能集。这可以使用社区创建的第三方库来完成。

<br />

<blockquote className="warning">
⚠️ &nbsp; 警告

免责声明：这些库可能存在错误、维护有限或其他限制，并非由 pmndrs 或 valtio 维护者正式推荐。此列表旨在为希望扩展 valtio 功能集的用户提供良好的起点。

</blockquote>

<br />

- [electron-valtio](https://github.com/water-a/electron-valtio) - 通过 valtio 在 Electron 主进程和各种渲染器窗口之间共享状态
- [eslint-plugin-valtio](https://github.com/pmndrs/eslint-plugin-valtio) - valtio 的 Eslint 插件
- [storybook-valtio-auto-bind](https://github.com/CosPie/storybook-valtio-auto-bind) - 自动将您的 Storybook 参数与 Valtio 存储双向同步
- [sveltio](https://github.com/wobsoriano/sveltio) - 使用代理的 Svelte 状态管理解决方案。由 valtio 提供支持。
- [swc-plugin-valtio](https://github.com/sosukesuzuki/swc-plugin-valtio) - Valtio useProxy 的 SWC 转换器。
- [tauri-plugin-valtio](https://github.com/ferreira-tb/tauri-store/tree/main/packages/plugin-valtio) - Tauri 的持久化 valtio 状态，可从 JavaScript 和 Rust 访问。
- [use-valtio](https://github.com/dai-shi/use-valtio) - 使用 Valtio 代理状态的另一个自定义 hook
- [valtio-element](https://github.com/lxsmnsyc/valtio-element) - 使用 valtio 创建响应式、声明式自定义元素
- [valtio-factory](https://github.com/mfellner/valtio-factory) - 使用工厂模式创建 valtio 状态
- [valtio-fsm](https://github.com/valtiojs/valtio-fsm) - 一个简单且可链接的 TypeScript 优先有限状态机库，由 Valtio 的响应式系统提供支持。
- [valtio-persist](https://github.com/valtiojs/valtio-persist) - 灵活且高性能地将状态保存到磁盘。
  - [valtio-auto-persist](https://github.com/valtiojs/valtio-auto-persist) - valtio-persist 的实验性分支，允许存储状态对象而无需附加键来标识它。它在幕后使用 [structure-id](https://github.com/overthemike/structure-id)。
- [valtio-plugin](https://github.com/valtiojs/valtio-plugin) - 一个全新的生命周期插件系统，使自定义使用 valtio 的方式变得更加容易。
- [valtio-reactive](https://github.com/valtiojs/valtio-reactive) - Valtio-reactive 使 valtio 成为一个响应式库。
- [valtio-signal](https://github.com/dai-shi/valtio-signal) - Valtio 代理状态的另一个 React 绑定
- [valtio-yjs](https://github.com/dai-shi/valtio-yjs) - Valtio-yjs 使 yjs 状态变得简单
- [valtio-zod](https://github.com/valtiojs/valtio-zod) - 使用 [Zod](https://zod.dev) 验证您的 valtio 状态更新
